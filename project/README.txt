Final Project, Option 2

1. The TM cycles through the three numbers for each digit in the longest of the first two numbers. On each cycle it makes sure that
the coressponding digit of the 3rd number (which is the number after the = sign) is what it should be. 
After reading a digit, it replaces that digit with a 'x' character to mark that its done with it. 
Its branches deal with both possible digits read from each of the first and second number, whether there is a carry or not, 
and when one of the have no remaining digits. 
In order to avoid false negatives, the TM input requires that the 3rd number has at least as many digits as 
the highest amount between the two other numbers.
The comments alongside the code explain what is happening for states where the turing machine has more than one
next state that it could go to depending on what the current tape symbol is. 

2. In my testing, its only errors were false negatives which occured in cases where the first and/or second number has more 
digits than the third number such as in "100+1=01" where it wrongly rejects because the turing machine is 
expecting a 0 at the end of the third number where there is instead a blank.
